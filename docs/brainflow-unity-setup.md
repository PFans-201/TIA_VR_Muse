# BrainFlow Unity Setup

Manual steps to install the BrainFlow plugin into Unity and enable the Muse S Athena
integration. These steps require clicking through Unity's GUI — they cannot be scripted.

## Overview

BrainFlow is distributed as a NuGet package (`brainflow`). Unity does not use NuGet
natively, so you extract the files manually and drop them into `Assets/Plugins/`.
The project uses a compile guard (`MUSE_BRAINFLOW`) so it builds and runs without
errors even when the plugin is not installed — the adapter just becomes a no-op stub.

## Step 1 — Download the NuGet package

1. Open a browser and go to: `https://www.nuget.org/packages/brainflow`
2. Click **Download package** — saves a `.nupkg` file (it is a ZIP archive).
3. Rename the extension from `.nupkg` to `.zip`.
4. Extract it somewhere temporary (e.g. `~/Downloads/brainflow_extracted/`).

Inside the extracted folder, the files you need are:

```
lib/
  netstandard2.0/
    brainflow.dll          ← managed C# assembly
    brainflow.xml          ← XML docs (optional)

build/native/               (or runtimes/)
  linux-x64/
    libBrainFlow.so
    libDataHandler.so
    libMLModule.so
  win-x64/
    BrainFlow.dll
    DataHandler.dll
    MLModule.dll
  osx-x64/
    libBrainFlow.dylib
    libDataHandler.dylib
    libMLModule.dylib
```

> The exact subfolder names vary between BrainFlow versions. Look for the `.so` / `.dll` /
> `.dylib` native files — they are typically under `runtimes/<rid>/native/` in newer versions.

## Step 2 — Copy files into Assets/Plugins/

Create this folder structure inside your Unity project:

```
Assets/
  Plugins/
    brainflow.dll                     ← the managed assembly (same file for all platforms)
    Linux/
      x86_64/
        libBrainFlow.so
        libDataHandler.so
        libMLModule.so
    Windows/
      x86_64/
        BrainFlow.dll
        DataHandler.dll
        MLModule.dll
    macOS/
      libBrainFlow.dylib
      libDataHandler.dylib
      libMLModule.dylib
```

Only copy the folders for the platforms you actually build for. At minimum you need Linux
(for development on Arch) or Windows depending on your workstation.

## Step 3 — Configure the native library import settings in Unity

Unity needs to know which platform each native library targets.

1. In the **Project** panel, navigate to `Assets/Plugins/Linux/x86_64/libBrainFlow.so`.
2. Select it. In the **Inspector** you will see **Plugin Import Settings**.
3. Uncheck **Any Platform**, then check only **Linux** under **Include Platforms**.
4. Set **CPU** to **x86_64**.
5. Click **Apply**.
6. Repeat for `libDataHandler.so` and `libMLModule.so`.

Do the same for the Windows DLLs (Platform = Windows, CPU = x86_64) and macOS dylibs
(Platform = macOS).

For `brainflow.dll` (the managed C# assembly):
- Select it in the Inspector.
- Ensure **Any Platform** is checked (it is platform-independent managed code).
- Click **Apply**.

## Step 4 — Add the MUSE_BRAINFLOW scripting define symbol

This unlocks the BrainFlow-specific code in `MuseAthenaAdapter.cs`. Without it the adapter
compiles as a harmless stub and all EEG features are silently disabled.

1. In the Unity menu bar: **Edit → Project Settings → Player**
2. Select the target platform tab (PC, Mac & Linux Standalone or Android for Quest builds).
3. Scroll to **Script Compilation** (sometimes labelled **Other Settings → Scripting Define Symbols**).
4. In the text field, add `MUSE_BRAINFLOW` (separated by a semicolon from any existing defines).
5. Click **Apply** (or press Enter — the Editor will recompile).

After recompilation, the Console should show no errors related to `brainflow`.

## Step 5 — Build all scenes

The Tutorial and Puzzle scenes are generated by an editor tool, not stored as committed scene files.

1. In the Unity menu bar: **Puzzle Game → Build All Scenes**
2. This creates (or re-creates):
   - `Assets/Scenes/TutorialRoom.unity`
   - `Assets/Scenes/ZenPuzzleRoom.unity`
   - `Assets/Scenes/EntryHall.unity`
3. Open **File → Build Settings** and confirm all three scenes appear in the **Scenes In Build**
   list in order: EntryHall (0), TutorialRoom (1), ZenPuzzleRoom (2).
   If they are missing, drag them in from the Project panel.

## Step 6 — Set the MAC address

1. Open the `TutorialRoom` scene.
2. In the Hierarchy, find the `MuseAthenaAdapter` GameObject.
3. In the Inspector, enter the MAC address of your Muse S Athena in the **Mac Address** field
   (format: `D4:22:CD:00:AA:BB`).
4. Save the scene (**Ctrl+S**).

> You can find the MAC address with `bluetoothctl scan on` (Linux) — see `docs/eeg-debug-guide.md`.

## Step 7 — Verify in Play Mode

1. Power on the Muse S Athena.
2. Press **Play** in the Unity Editor.
3. Open the **Console** window. Within a few seconds you should see:
   ```
   [MuseAthenaAdapter] Connected to Muse S Athena.
   [MuseAthenaAdapter] Baseline phase started (60 s)…
   ```
4. After the baseline phase completes:
   ```
   [MuseAthenaAdapter] Baseline complete. θ=0.31 ± 0.04  α=0.57 ± 0.06
   ```

If you see `[MuseAthenaAdapter] MUSE_BRAINFLOW not defined — running as stub.` the
scripting define symbol was not applied correctly. Return to Step 4.

## Troubleshooting

### "The type or namespace 'brainflow' could not be found"

The managed `brainflow.dll` is either missing from `Assets/Plugins/` or its import settings
have **Any Platform** unchecked. Verify the file is present and re-apply import settings.

### "DllNotFoundException: BrainFlow" at runtime

The native library for your platform is missing or its import settings point to the wrong
platform/CPU combination. Re-check Step 3 for the native `.so` / `.dll` files.

### Unity does not show "Plugin Import Settings" for a .so file

Make sure the file is inside `Assets/Plugins/` (or a subdirectory). Files outside
`Assets/Plugins/` are not treated as plugins by Unity.

### Build for Android / Quest

BrainFlow supports Android (arm64). In the NuGet package look for `android-arm64` native
libraries. Place them in `Assets/Plugins/Android/` and set Platform = Android, CPU = ARM64
in the Import Settings. The BrainFlow board connection over Bluetooth works the same way.

## Notes on version compatibility

BrainFlow 5.x introduced `MUSE_S_ATHENA_BOARD` (board ID 67). If you have an older
version installed and the project fails to compile, check the BrainFlow changelog and
update to 5.x or later.
